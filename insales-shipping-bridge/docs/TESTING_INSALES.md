# Как протестировать приложение на тестовом магазине inSales

## Важно: кабинет разработчика ≠ магазин

**Аккаунт партнёра/разработчика** (где вы создаёте приложение) — это не интернет-магазин.  
Тестовый магазин — **отдельная регистрация** витрины `*.myinsales.ru`. Без него приложение некуда установить.

---

## Шаг 1. Создать тестовый магазин

1. Откройте [https://www.insales.ru/](https://www.insales.ru/) в режиме инкогнито (или с другим email, не тем что у партнёрского кабинета).
2. Нажмите **«Создать магазин»** / **«Попробовать бесплатно»** — триал обычно 14 дней.
3. Заполните название, email, пароль.
4. После регистрации запомните адрес магазина: **`ваш-магазин.myinsales.ru`**.
5. Войдите в **админку** магазина (не в кабинет разработчика).

> Если магазин уже есть (ваш или клиента) — можно тестировать на нём: установка приложения доступна любому магазину, который установит ваше приложение.

---

## Шаг 2. Приложение в кабинете разработчика

1. Войдите в **кабинет партнёра inSales** (раздел приложений / API).
2. Создайте приложение или откройте существующее.
3. Укажите URL (ваш сервер, корень `public`):

   | Поле | Пример |
   |------|--------|
   | URL установки | `http://176.53.160.91/insales/install` |
   | URL входа | `http://176.53.160.91/insales/app` |
   | URL удаления | `http://176.53.160.91/insales/uninstall` |

4. Скопируйте **ID приложения** и **секрет** → в `.env` на сервере:
   - `INSALES_APP_ID`
   - `INSALES_APP_SECRET`
5. Также на сервере: `SHIPPING_API_APPKEY`, логин/пароль API перевозчика, MySQL.

Для продакшена inSales рекомендует **HTTPS**; на IP для первых тестов часто достаточно HTTP.

---

## Шаг 3. Установить приложение на тестовый магазин

Ссылки `/admin/applications/dellin_ship_AnY/install` **часто не работают** для неопубликованных приложений — это нормально.

### Ручная установка (рекомендуется)

1. На сервере в `.env`: `MANUAL_INSTALL_SECRET=любая-длинная-строка`
2. В админке магазина откройте: `https://myshop-ddy891.myinsales.ru/admin/account.json` — скопируйте `"id"`
3. Откройте: `http://176.53.160.91/insales/manual-install?secret=ВАШ_СЕКРЕТ`
4. Укажите shop и insales_id → **Установить** → настройки `/insales/app`

---

## Шаг 4. Проверка без витрины

На сервере (подставьте хост магазина):

```bash
curl -s "http://127.0.0.1/insales/terminals?city_kladr=77&limit=3"
curl -s -X POST "http://127.0.0.1/v1/calculate-from-variants" \
  -H "Content-Type: application/json" \
  -H "X-Bridge-Token: ВАШ_СЕКРЕТ" \
  -d '{"shop":"ваш-магазин.myinsales.ru","lines":[{"variant_id":1,"product_id":1,"quantity":1}],"arrival_terminal_id":12345}'
```

`variant_id` / `product_id` возьмите из админки: **Товары** → карточка → вариант.

Карта ПВЗ:  
`http://176.53.160.91/widget/index.html?bridge=http://176.53.160.91`

---

## Шаг 5. Checkout

В админке **магазина** настройте **внешний способ доставки** (API v2) — URL вашего бриджа.  
Инструкция: [CHECKOUT_INSALES.md](CHECKOUT_INSALES.md). Это не платная «Доставка inSales».

## Миграция БД (новые поля настроек)

```bash
cd /var/www/insales-integration
mysql -u bridge_user insales_bridge < ./insales-shipping-bridge/database/migrations/003_shop_delivery_settings.sql
```

## Частые проблемы

| Симптом | Решение |
|---------|---------|
| Нет пункта «Приложения» в админке | Убедитесь, что зашли в **магазин**, а не в кабинет разработчика |
| Ошибка при установке | Проверьте URL приложения, MySQL, `INSALES_APP_SECRET` |
| Пустой список терминалов | `SHIPPING_API_APPKEY` в `.env`, права на `var/cache/` |
| `could not find driver` | Установите `php-mysql` / `mariadb-server`, перезапустите php-fpm |
